## 5. CF Redirect Rule（待用户手动 3 分钟）

CF dashboard 在 CDP 监控下被反自动化拦截（splash 卡死，5/8 也遇到过且这次更严），
无法用 web-access skill 自动建规则。机器上也没有 CLOUDFLARE_API_TOKEN 凭据。

最快路径：用户在自己 Chrome 手动建 Single Redirect：

1. https://dash.cloudflare.com → 选 aierxuanlaptop.com zone
2. 左侧 Rules → Redirect Rules → Create rule
3. **Rule name**: `apex-to-www-301`
4. **When incoming requests match → Custom filter expression**
   - Field: `Hostname`  Operator: `equals`  Value: `aierxuanlaptop.com`
5. **Then → Type**: `Dynamic`
6. **Expression**: `concat("https://www.aierxuanlaptop.com", http.request.uri.path)`
7. **Status code**: `301`
8. ✅ **Preserve query string**: 勾选
9. Deploy

验证：`curl -sI https://aierxuanlaptop.com/en` 应返回 `HTTP/2 301` +
`location: https://www.aierxuanlaptop.com/en`。

## 6. /uploads 真 1y immutable（缓做）

CF 已 HIT 30 天，性价比不高，等下次 VPS 重启加载 kernel 时顺手在
`/etc/nginx/sites-enabled/aierxuanlaptop.com` 加：
```
location ~ ^/(images|uploads)/ {
  add_header Cache-Control "public, max-age=31536000, immutable";
}
```
然后 `nginx -t && nginx -s reload`。
