### 2.4 技术问题修复

#### Yandex HTML 文件验证问题（已解决）
- **问题**: Next.js 将 `/yandex_*.html` 路由到主页而非静态文件
- **原因**: Next.js middleware 拦截了所有路由
- **解决方案**: 改用 DNS 验证方式（更可靠）

#### 相关代码修改（供参考）
```typescript
// middleware.ts - 添加排除规则
export const config = {
    matcher: ['/((?!api|admin|_next/static|_next/image|images|uploads|favicon.ico|icon.svg|sitemap.xml|robots.txt|yandex_[^/]+\\.html).*)'],
}

// layout.tsx - 添加 Meta 标签验证
verification: {
  yandex: 'cae9fe4eb8159f38',
},
```
