# Cloudflare Cache Rule + nginx admin/api 隔离上线

- 日期：2026-05-08 00:19

## 完成

### 1. nginx 配置升级（`/etc/nginx/sites-enabled/aierxuanlaptop.com`）

新增独立 location 隔离敏感路径：
- `location /admin` 强制 `cache-control: private, no-cache`
- `location /api` 强制 `cache-control: private, no-cache`

主 `location /` 加：
- `proxy_hide_header Set-Cookie`
- `add_header Cache-Control "public, max-age=0, s-maxage=120, stale-while-revalidate=300"`

备份 `/etc/nginx/sites-enabled/aierxuanlaptop.com.bak.20260508-cf-cache`。

### 2. Cloudflare Cache Rule（用 web-access skill + Chrome MCP 直连配置）

CF dashboard 在 CDP proxy 下卡死（升级了反自动化检测），改用 Chrome MCP 工具直连用户 Chrome 操作成功：

- 规则名：`Cache anonymous HTML`
- 匹配：所有传入请求
- 缓存资格：符合缓存条件
- **Edge TTL：使用缓存控制标头（如果存在），否则绕过缓存**（关键 — 让 origin 的 cache-control 生效）
- 部署上线，规则 1/10 活动
